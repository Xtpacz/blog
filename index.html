类加载机制

Java虚拟机的类加载，是整个Java程序运行的基础（不加载怎么用哇）

这个机制负责将字节码文件（.class）转化为内存中的Java对象，并进行链接？和初始化？。

过程分为五个阶段：

  1. 加载
  2. 验证
  3. 准备
  4. 解析（resolution）
  5. 初始化

加载（拿到）

通过ClassLoader读取字节码文件（User.class）。
将读取到的字节码文件转化为java.lang.Class对象，存储在方法区。
创建一个指向该Class对象的应用，放在堆内存中。

比如，当调用User user = new User()时，JVM首先会检查User类是否已经加载。
如果没有加载，则启动类加载器读取字节码文件（User.class）

验证（检查）

为了防止恶意或错误的字节码破坏JVM，因此要加载出来的字节码进行检查。

检查以下几个方面：

  1. 文件格式：是否符合Class文件规范：魔数0xCAFEBABE，版本号是否兼容。。
  2. 元数据：类的继承关系是否正确（如是否继承final类
  3. 字节码：确保代码逻辑合法（如是否有非法跳转指令
  4. 符号引用：检查符号引用是否存在（如引用的类是否加载

准备

为类变量（static变量）分配内存，并设置初始值（int：0，引用：null）
此时不会执行任何Java代码，初始值是由JVM设定的。

解析

将符号引替换为直接引用：
比如将类名、方法名字节替换为内存地址，方法偏移量。

符号引用：以文本形式存在的引用，比如com.example.User.getName
直接引用：指向内存中具体位置的指针。

Kerberos学习笔记

参考: https://www.geeksforgeeks.org/kerberos/

Kerberos是一种网络认证协议。源于希腊神话中的Cerberus，由MIT开发。
分为三个角色：

  1. Client
  2. Server
  3. KDC - 第三方的可信服务器 Key Distribution Center

Kerberos的三个主要组成部分：

  • Authentication Server (AS): The Authentication Server performs the initial authentication and ticket for Ticket Granting Service.
  • Database: The Authentication Server verifies the access rights of users in the database.
  • Ticket Granting Server (TGS): The Ticket Granting Server issues the ticket for the Server

工作流程-三阶段认证

  1. 第一阶段：获取票据授予票据（TGT, Ticket Granting Ticket）

    1. 客户端向 AS 发送包含用户名的认证请求。
    2. AS 验证用户身份后，用用户密码对应的密钥加密生成 TGT，并返回给客户端。

  2. 第二阶段：获取服务票据

    1. 客户端使用 TGT 向 TGS 请求访问特定服务的票据。
    2. TGS 验证 TGT 有效性后，生成服务票据（包含客户端身份和会话密钥），返回给客户端。

  3. 第三阶段：服务认证

    1. 客户端向服务服务器发送服务票据和认证信息。
    2. 服务器验证票据有效性后，建立安全连接并提供服务。

在大数据领域应用比较广泛。
